**1. Main Table: `sdgs`**

* **`goal_id` (INT, PRIMARY KEY):** Unique identifier for each SDG (e.g., 1, 2, 3, ... 17).
* **`goal_name` (VARCHAR):** The name of the SDG (e.g., "No Poverty", "Zero Hunger").
* **`goal_description` (TEXT):** A detailed description of the SDG's objectives.
* **`target_year` (INT):** The year by which the SDG aims to be achieved (e.g., 2030).

**2. Table: `targets`**

* **`target_id` (INT, PRIMARY KEY):** Unique identifier for each target within an SDG.
* **`goal_id` (INT, FOREIGN KEY REFERENCES `sdgs`(`goal_id`)):** Links the target to its corresponding SDG.
* **`target_description` (TEXT):** A detailed description of the specific target.
* **`target_indicator` (VARCHAR):** A specific indicator used to measure progress towards the target.

**3. Table: `indicators`**

* **`indicator_id` (INT, PRIMARY KEY):** Unique identifier for each indicator.
* **`target_id` (INT, FOREIGN KEY REFERENCES `targets`(`target_id`)):** Links the indicator to its corresponding target.
* **`indicator_name` (VARCHAR):** The name of the indicator (e.g., "Proportion of population living below the international poverty line").
* **`indicator_definition` (TEXT):** A detailed definition of how the indicator is measured.
* **`data_source` (VARCHAR):** The source of data for the indicator (e.g., World Bank, UN Data).
* **`data_type` (VARCHAR):** The type of data (e.g., percentage, number, ratio).

**4. Table: `progress`**

* **`progress_id` (INT, PRIMARY KEY):** Unique identifier for each progress record.
* **`indicator_id` (INT, FOREIGN KEY REFERENCES `indicators`(`indicator_id`)):** Links the progress record to its corresponding indicator.
* **`year` (INT):** The year for which the progress data is recorded.
* **`value` (DECIMAL):** The actual value of the indicator for that year.

**5. Table: `countries`**

* **`country_id` (INT, PRIMARY KEY):** Unique identifier for each country.
* **`country_name` (VARCHAR):** The name of the country.
* **`iso_code` (VARCHAR):** The ISO 3166-1 alpha-3 code for the country.

**6. Table: `country_progress`**

* **`country_progress_id` (INT, PRIMARY KEY):** Unique identifier for each country's progress record.
* **`country_id` (INT, FOREIGN KEY REFERENCES `countries`(`country_id`)):** Links the progress record to its corresponding country.
* **`indicator_id` (INT, FOREIGN KEY REFERENCES `indicators`(`indicator_id`)):** Links the progress record to its corresponding indicator.
* **`year` (INT):** The year for which the progress data is recorded.
* **`value` (DECIMAL):** The actual value of the indicator for that country and year.

**Relationships:**

* **`sdgs`** has a one-to-many relationship with **`targets`**.
* **`targets`** has a one-to-many relationship with **`indicators`**.
* **`indicators`** has a one-to-many relationship with **`progress`**.
* **`countries`** has a many-to-many relationship with **`indicators`** through **`country_progress`**.

**Benefits of this Structure:**

* **Organized Data:**  The database effectively separates and organizes information about SDGs, targets, indicators, and progress.
* **Flexibility:**  The structure allows for easy addition of new SDGs, targets, indicators, and progress data as needed.
* **Analysis:**  The database facilitates analysis of progress towards each SDG, both globally and at the country level.
